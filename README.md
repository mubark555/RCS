# نظام إدارة مشروع سيم برايم (Seem Prime)

نظام إلكتروني متكامل لإدارة مشاريع ومهام المتعاقدين مع الشريك التشغيلي **فيوليت (Violet)**.
بديل رقمي كامل عن ملف الإكسل — مبني بـ **Next.js** وقاعدة بيانات سحابية **Supabase**.

## المميزات

- **📊 لوحة مؤشرات:** نسبة الإنجاز، صحة المهام (سليمة / معرّضة للتعثر / متعثرة)، التقدّم حسب المشروع.
- **📋 إدارة المهام:** إضافة/تعديل/حذف، بحث وفلترة حسب المشروع والحالة — بكل حقول الإكسل
  (النشاط، الأولوية، الحالة، المسؤول، الاستحقاق، بانتظار، العائق، حالة الاعتماد، الصحة، ملاحظات).
- **📅 جدولة الاجتماعات:** عنوان، تاريخ ووقت، مدة، مكان/رابط، حضور، جدول أعمال، حالة.
- **🗄️ أرشيف الملفات:** رفع الملفات وتصنيفها حسب المشروع/النوع، والاطلاع عليها وتنزيلها.
- **☁️ قاعدة بيانات مشتركة** عبر Supabase — يستخدمها الفريق كامل من أي جهاز.
- تم **نقل الـ 91 مهمة** من ملف الإكسل الأصلي كبيانات جاهزة.

## وضعان للتشغيل

| الوضع | متى يعمل | الوصف |
| --- | --- | --- |
| **محلي (تجريبي)** | إذا لم تُضبط مفاتيح Supabase | يعمل فوراً، البيانات تُحفظ في متصفحك فقط (localStorage). مناسب للتجربة. |
| **سحابي** | عند ضبط مفاتيح Supabase | قاعدة بيانات وتخزين مشترك للفريق كامل. |

يظهر الوضع الحالي كشارة أعلى يمين الشاشة (◐ محلي / ☁ سحابي).

## التشغيل محلياً

```bash
npm install
npm run dev
```

ثم افتح <http://localhost:3000>. سيعمل مباشرة في الوضع المحلي التجريبي مع الـ 91 مهمة.

## ربط قاعدة البيانات السحابية (Supabase)

1. أنشئ مشروعاً مجانياً على [supabase.com](https://supabase.com).
2. من **SQL Editor** شغّل الملفين بالترتيب:
   - `supabase/schema.sql` (إنشاء الجداول + التخزين + الصلاحيات)
   - `supabase/seed.sql` (تعبئة الـ 91 مهمة)
3. من **Project Settings → API** انسخ:
   - `Project URL`
   - `anon public key`
4. أنشئ ملف `.env.local` (انسخ من `.env.example`):

   ```bash
   NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOi...
   ```

5. أعد التشغيل `npm run dev` — ستتحول الشارة إلى **☁ سحابي** ويشارك الفريق نفس البيانات.

> **ملاحظة أمان:** ملف `schema.sql` يضع سياسات وصول مفتوحة للبدء السريع. للاستخدام الحقيقي
> فعّل مصادقة Supabase (Auth) وقيّد السياسات حسب المستخدم.

## النشر على Vercel

1. ارفع المستودع إلى GitHub.
2. من [vercel.com](https://vercel.com) اختر **New Project** واربطه بالمستودع.
3. أضف متغيّري البيئة `NEXT_PUBLIC_SUPABASE_URL` و `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
4. اضغط **Deploy**.

## بنية المشروع

```
app/
  page.js            لوحة المؤشرات
  tasks/page.js      إدارة المهام
  meetings/page.js   جدولة الاجتماعات
  archive/page.js    أرشيف الملفات
  layout.js          الهيكل العام (RTL عربي) + التنقّل
  globals.css        التنسيق
components/           مكوّنات واجهة مشتركة (NavBar, Modal, Badge, TaskForm)
lib/
  supabase.js        عميل Supabase + كشف الوضع
  store.js           طبقة البيانات (سحابي / محلي)
  constants.js       القوائم والحالات والألوان
  seed-data.js       الـ 91 مهمة المستوردة من الإكسل
supabase/
  schema.sql         مخطط قاعدة البيانات
  seed.sql           بيانات الـ 91 مهمة
scripts/
  generate-seed.mjs  أداة توليد seed.sql
```
